# ROS-APP 更新检测与发布说明

## 1. 当前实现概览

- 客户端更新检测入口在“关于”页，按钮事件：`OnCheckUpdateClicked`。
- 更新源地址已写死为：
  - `http://ros.yuban.cloud/ros/latest.json`
- 客户端只负责：
  - 拉取 `latest.json`
  - 比较版本号
  - 展示更新说明
  - 打开下载链接

关键代码位置：
- `ROS-APP/Views/MainPage.xaml.cs`
  - `FixedUpdateFeedUrl`
  - `ParseUpdateInfo`
  - `CompareVersion`
  - `OnCheckUpdateClicked`
  - `OnOpenUpdateUrlClicked`

## 2. latest.json 规范

当前发布脚本输出的标准字段：

```json
{
  "version": "0.0.48",
  "downloadUrl": "http://ros.yuban.cloud/ros/beta/ROS-APP-Setup-0.0.48.exe",
  "notes": "beta版：UI交互优化打包",
  "sha256": "8C2A174E622B0703DC3B9CD0801940B7212166B766BE264CC01C4FCCAE08D129",
  "buildTime": "2026-02-23T16:19:46+08:00"
}
```

客户端解析兼容字段（向后兼容）：
- 版本：`version` / `latestVersion` / `tag_name`
- 下载地址：`downloadUrl` / `url` / `html_url` / `assets[0].browser_download_url`
- 更新说明：`notes` / `body`

## 3. 版本比较规则

- 使用语义版本比较（如 `0.0.48 > 0.0.47`）。
- 支持 `v0.0.48` 形式（会自动去掉前缀 `v`）。
- 若无法解析为标准版本，则回退到字符串比较。

## 4. 双通道（beta/prod）发布结构

- 产物目录
  - `dist/beta/ROS-APP_x.y.z`
  - `dist/prod/ROS-APP_x.y.z`
- 输出目录
  - `out/beta/ROS-APP-Setup-x.y.z.exe`
  - `out/beta/latest.json`
  - `out/prod/ROS-APP-Setup-x.y.z.exe`
  - `out/prod/latest.json`
- 变更日志
  - `CHANGELOG.beta.txt`
  - `CHANGELOG.prod.txt`
  - `CHANGELOG.txt`（汇总）

## 5. 发布脚本职责

### 5.1 `scripts/auto_build.ps1`

- 自动递增项目版本号（补丁位 +1）。
- 按 `-Channel beta|prod` 输出到对应 `dist/<channel>`。
- 写入对应通道日志：`CHANGELOG.<channel>.txt`。
- 默认会继续调用 `build_exe_installer.ps1` 产出安装包和 `latest.json`。

常用命令（beta）：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/auto_build.ps1 -Channel beta -Environment beta -Message "本次更新说明"
```

常用命令（prod）：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/auto_build.ps1 -Channel prod -Environment prod -Message "正式版发布说明"
```

### 5.2 `scripts/build_exe_installer.ps1`

- 从 `dist/<channel>/ROS-APP_x.y.z` 读取程序目录。
- 打包安装器到 `out/<channel>`。
- 生成 `out/<channel>/latest.json`。
- `notes` 自动从 `CHANGELOG.<channel>.txt` 对应版本行提取。

### 5.3 `scripts/sync_beta_to_prod.ps1`

- 将指定（或最新）beta版本目录同步到 `dist/prod`。
- 将 beta 日志（截至该版本）补齐到 `CHANGELOG.prod.txt`（去重）。
- 最后自动执行 prod 安装包与 `latest.json` 发布。

常用命令：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/sync_beta_to_prod.ps1 -Version 0.0.48
```

## 6. 服务器侧部署要求

- `prod` 更新地址固定为：`http://ros.yuban.cloud/ros/latest.json`
- 推荐同时提供：
  - `http://ros.yuban.cloud/ros/ROS-APP-Setup-x.y.z.exe`
  - `http://ros.yuban.cloud/ros/latest.json`
- beta 可独立放在：
  - `http://ros.yuban.cloud/ros/beta/ROS-APP-Setup-x.y.z.exe`
  - `http://ros.yuban.cloud/ros/beta/latest.json`

## 7. 当前限制与建议

- 客户端更新源目前固定到 prod：`FixedUpdateFeedUrl`。
  - 这意味着 beta 客户端默认也检查 prod 更新源。
- 若后续需要“beta 客户端检查 beta 更新”，建议增加：
  - 设置项：更新通道（beta/prod）
  - 或在编译时注入不同 `FixedUpdateFeedUrl`

## 8. 快速排查

- 点击“检查更新”无响应：
  - 先看日志区是否有 `开始检查更新`。
- 提示“更新源缺少 version/tag_name 字段”：
  - 检查 `latest.json` 是否包含 `version`（或兼容字段）。
- 提示“当前没有可用下载地址”：
  - 检查 `latest.json` 是否包含 `downloadUrl`（或兼容字段）。
- 版本不生效：
  - 检查 `ROS-APP.csproj` 的 `<Version>` 是否被构建脚本成功递增。
